# How-to guide template

Title pattern: "How to <specific user goal>" — the title states exactly what the guide
achieves, in the user's vocabulary, never the product's internal feature name.
(Directions for a competent user at work. The reader owns the goal; the guide supplies
the moves.)

---

# How to <goal>

This guide shows you how to <goal, one sentence>. <Optional single sentence scoping:
what situation this applies to.>

<Optional "Before you start" — only genuinely non-obvious preconditions. Rely on
implicit competence for the basics.>

## <First meaningful stage, named by sub-goal>

1. <Imperative step.>
2. <Imperative step. Conditionals are fine: "If you want x, do y.">

## <Next stage>

1. <Steps in the order a person would actually think and act.>

<Point outward instead of inlining detail: "For the full list of <options>, see
[<reference page>].">

---

Checklist before publishing:
- [ ] Title names the exact task ("How to …")
- [ ] Organized around the user's goal, not product structure
- [ ] Action only — no teaching, no background (link to explanation if needed)
- [ ] Starts and ends at sensible points (usable beats exhaustive)
- [ ] Exhaustive detail delegated to reference pages by link
